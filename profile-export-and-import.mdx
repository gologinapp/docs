This guide explains how to export and import profiles in Gologin and when to use each method.

Gologin supports profile import and export via CSV files.

## What is profile export and import

Profile export and import allow you to:

- copy profile configurations
- create multiple profiles at once
- edit and update profiles in bulk
- move profile setups between workspaces

All operations are performed using a **CSV file (comma-separated values)**.

## What is included

CSV import/export works with profile configuration only.

Included:

- profile name
- proxy settings
- folders and tags
- basic environment settings

Not included:

- cookies
- sessions
- login state
- full fingerprint identity

## How to export profiles

1. Select one or more profiles
2. Click **Export profiles**
3. Save the CSV file

## How to import profiles

Profiles can be imported from the workspace menu.

1. Open the top-right menu
2. Click **Import profiles**
3. Upload a CSV file or paste CSV data
4. Click **Import**

## When to use CSV import/export

Use this method if you need to:

- create many profiles quickly
- manage proxies across multiple profiles
- standardize team setups
- automate profile creation

## Important limitations

### Imported profiles are new

Profiles created via CSV:

- do not contain previous sessions
- are not logged in
- behave like new environments

### CSV is not a backup

Exporting profiles does NOT include:

- cookies
- account logins
- browsing history

### CSV requires correct format

Incorrect CSV structure may:

- break import
- create invalid profiles

Use the template format when possible.

## Common mistake

CSV import/export is often confused with full profile transfer.

 It does NOT transfer:

- logged-in accounts
- cookies
- full browser identity

## Key takeaway

CSV import/export is used for **profile setup and scaling**, not for transferring profile identity.
