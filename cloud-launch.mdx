---
title: "Cloud Launch"
---

Cloud Launch allows you to run **Gologin browser profiles on remote cloud servers** instead of your local device.

When Cloud Launch is used, the browser environment runs in the cloud while you interact with it through the Gologin interface. This allows profiles to operate independently from your local machine.

Cloud Launch can be useful when working remotely or when running profiles without relying on local hardware resources.

## How Cloud Launch works

1. A browser profile is launched on a remote cloud server.
2. The browser environment runs on the server instead of your local computer.
3. You connect to the running profile through the Gologin interface.
4. All browsing activity happens inside the cloud environment.

Because the browser runs remotely, local device resources such as CPU or RAM are not used for browser execution.

## Benefits of Cloud Launch

Using Cloud Launch can help:

- run profiles without using local system resources
- access profiles from different devices
- keep profile sessions independent from your local machine
- support remote workflows for teams

## Limitations

When using Cloud Launch:

- a stable internet connection is required
- performance may depend on network latency
- profiles should not be launched simultaneously from multiple environments

## Best use cases

Cloud Launch can be useful for:

- remote work with browser profiles
- running profiles on devices with limited resources
- distributed teams accessing shared profiles
- maintaining sessions independent of a specific device
